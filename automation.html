About This Module

This module is the entry point to Resolve's automation learning program — designed to help IT teams move from "we should automate something" to a concrete first candidate with defined success criteria, in a single session.

The central challenge in automation adoption isn't technical. Most teams understand what automation can do. The block is knowing where to start — and building enough internal confidence to move forward without waiting for a perfect use case. This module is built around that problem.

Design Decisions
  • The Automatable persona exists because abstract change management advice doesn't stick. A named company moving through the same journey as the learner creates emotional continuity across the program — learners follow Automatable's progress rather than consuming isolated modules.
  • The start-small framework (Choose → Define Success → Measure → Celebrate) is deliberately low-stakes. It's not a full automation methodology — it's a wedge. The goal is a single successful automation that builds internal credibility for the broader program.
  • Measurable outcomes are baked in from step one because adoption fails when teams can't demonstrate ROI. Defining success criteria before automating is an ID decision, not just a business one — it gives learners a completion condition for the activity.
  • RANI, the embedded AI assistant, is positioned inside the module itself with pre-written prompt suggestions. This models AI-assisted workflow adoption in real time — learners aren't just reading about AI tools, they're using one to complete the learning activity.
