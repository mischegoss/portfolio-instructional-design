ARIA Integration — System Overview
Conceptual overview  ·  Reading time: 4 minutes  ·  Audience: New integration developers
Learning Objective

Understand how ARIA connects a chat interface to a workflow automation platform — so users can trigger backend processes through conversation and get results without leaving their chat window.

Prerequisites

Familiarity with the following is helpful but not required:

  • APIs — Systems exchange data through structured requests and responses
  • Webhooks — Event-driven HTTP requests that push data between systems
  • JSON — Structured data format for API communication
  • Parameters — Discrete pieces of information collected from users
What This Integration Accomplishes

ARIA connects HelixBot (a conversational AI platform) to FlowEngine (a workflow automation platform). A user types a request in chat, HelixBot collects what it needs, and FlowEngine runs the workflow — all without the user touching another system.

Key Capability
Users interact with enterprise systems using natural language in HelixBot, without leaving the chat window.
How HelixBot and FlowEngine Work Together

HelixBot is the front end. It interprets what the user is asking, asks any follow-up questions, and packages the user's input into a structured request.

FlowEngine is the back end. It receives that request, runs the workflow, and sends the result back to HelixBot to display in chat.

The two systems talk to each other via webhook. HelixBot sends a JSON payload to FlowEngine's API, holds the conversation open, and displays whatever FlowEngine returns. The user sees only the result — the plumbing is invisible.

Bi-directional flow — one user request, two webhook calls between HelixBot and FlowEngine.
The Complete Flow

What happens when a user types "I need to reset my password":

# Stage What happens
1 User Trigger The user types a request. HelixBot's intent prediction matches it to the "password reset" workflow.
2 Information Gathering HelixBot collects required data through conversation or user profile data. Answers become the input parameters sent to FlowEngine.
3 Data Transformation HelixBot packages all inputs and session context into the JSON format FlowEngine expects. A mapping script handles the translation automatically — it ships with the pre-built templates, so you don't write it yourself.
4 Webhook Sent HelixBot sends the structured payload to FlowEngine's Events API endpoint and holds the conversation open.
5 Workflow Execution FlowEngine receives the request, routes it to the correct workflow, executes it, and packages the result.
6 Callback & Display FlowEngine sends the result back via the callback URL. The response appears in the user's chat: ✓ Password reset successfully.

Total time: typically 5–15 seconds from start to finish.

What You Can Build

This integration supports any workflow that collects structured input through conversation, executes a FlowEngine workflow, and returns a result to the user. Common use cases include password resets and account unlocks, software access requests, approval workflows, and system status checks.

Clone and customize the pre-built templates for each new use case. Most implementations take 30–60 minutes per workflow, and the consistent template structure ensures every integration behaves the same way.

Configure the ITSM Platform Reverse Connection
Procedural guide  ·  Estimated time: 15–20 minutes  ·  Audience: ITSM Platform admins
Context: This second sample covers a different integration in the same playbook. The ITSM Platform here is a separate back-end system from FlowEngine — HelixBot is the common front end, but this guide wires it to a ticketing system rather than a workflow engine.

At this point, HelixBot can send requests to the ITSM Platform. This guide sets up the other direction — so the ITSM Platform can send data back to HelixBot. You'll work entirely inside the ITSM Platform interface. Before you start, confirm that the prior setup step (committing the Helix update set) is complete.

Prerequisites
  • HelixBot configuration completed
  • Helix update set committed in the ITSM Platform
  • ITSM integration user password on hand
  • HelixBot tenant URL on hand
Steps
Step 1 Log in to the ITSM Platform
Important
All steps below happen in the ITSM Platform — not in HelixBot.
  1. Open your ITSM Platform instance in a browser.
  2. URL format: https://[instance].itsm-platform.com
  3. Log in with your ITSM Platform admin credentials.
Step 2 Open the Helix Integration Settings module
Check your application scope first
If you're in the wrong application scope, the Helix fields will not appear in the form — and you won't be able to save settings. Before proceeding, look for a scope banner at the top of the ITSM Platform and switch to the Helix application scope if prompted.
  1. In the ITSM Platform Navigator, type: Helix
  2. Locate the Helix application modules in the results.
  3. Click Helix > Integration Settings.

You should see a form titled "Integration Settings."

Step 3 Enter HelixBot connection credentials
Case sensitive
All three values below are case-sensitive. Copy them exactly as shown — including lowercase, punctuation, and the trailing slash on the URL.
Field Value
Username helix.integration
Password The integration user password from your prior setup step
Base URL See copyable format below
Base URL format https://{tenant}.helix-platform.com/
Don't forget the trailing slash
A missing trailing slash is the most common cause of connection failure.
✓ https://acme.helix-platform.com/
✗ https://acme.helix-platform.com

Click Save or Submit. Settings should save without errors.

Step 4 Open the Helix Integration Hub module
  1. In the ITSM Platform Navigator, type: Helix
  2. Click Helix > Integration Hub.

You should see a Test Connection or Test/Refresh Connection button.

Step 5 Test the connection
  1. Click Test/Refresh Connection.
  2. Wait 5–10 seconds.
  3. Look for a success message in the interface.
Bidirectional connection complete. HelixBot → ITSM Platform and ITSM Platform → HelixBot. The two systems can now communicate bidirectionally for ticket creation and data exchange.
Troubleshooting
Connection test fails — Missing roles
Cause: Integration user missing required roles
  • Verify all three Helix roles were assigned to the integration user.
  • Confirm the user also has the itil role if required by your instance.
  • Verify Password needs reset is unchecked on the user record.
Connection test fails — Authentication error
Cause: Incorrect username or password
  • Verify the username is exactly: helix.integration
  • Verify the password matches what was set during integration user setup.
  • Verify Password needs reset is unchecked.
Connection test fails — URL error
Cause: Missing trailing slash
  • The URL must include https:// and end with a trailing slash.
✓ https://acme.helix-platform.com/
✗ https://acme.helix-platform.com
Connection test fails — Network error
Cause: Firewall or network configuration
  • Verify the ITSM Platform instance can reach external URLs.
  • Contact your network admin if needed.
No response from test
Cause: HelixBot tenant unreachable
  • Verify the HelixBot tenant is accessible by opening it directly in a browser.
  • Wait 1–2 minutes and try the test again.
Helix modules not visible
Cause: Update set not committed or browser cache
  • Verify the update set status shows Committed.
  • Hard refresh your browser (Ctrl+Shift+R).
  • Log out and back in to the ITSM Platform.
